# Pixels et taille d'une image - Exercice

<div class="imported-content">

<p>Commencez par sauvegarder le fichier "Maison taguee image au format BMP.bmp" sur un ordinateur.<br/></p>
<p>Visualisez ses propriétés. Par exemple, sous Windows, cliquez avec le bouton droit de la souris sur le fichier, puis cliquez sur Propriétés. </p>
<p>1. Dans ces propriétés figurent le nombre de pixels de l'image. Notez ce nombre de pixels, en largeur et en hauteur. Notez aussi la taille en mémoire de l'image, en mégaoctets (Mo) qui est également indiquée dans les propriétés du fichier. <br/></p>
<p>2. Calculez le nombre total de pixels auquel cela correspond.<br/></p>
<p>La couleur de chaque pixel est codée par trois octets, chacun compris entre 0 et 255 : </p>
<ul><li>un octet pour le rouge (0 : pas de rouge ; 255 : intensité maximale de rouge)<br/></li><li>un octet pour le vert (0 : pas de vert ; 255 : intensité maximale de vert)<br/></li><li>un octet pour le bleu (0 : pas de bleu ; 255 : intensité maximale de bleu). <br/></li></ul><p>3. Calculez le nombre d'octets qui sont nécessaires pour coder une image de cette taille. </p>
<p>4. À combien de millions d'octets cela correspond-il ? Comparez ce résultat à la taille de l'image sur le disque dur. Vous devriez approximativement retrouver la même valeur. </p>
<p>Aide : un mégaoctet (1 Mo) représente environ un million d'octets. Mais plus rigoureusement, 1 Mo = 1024 Ko, et 1 Ko = 1024 octets. Les informaticiens préfèrent des multiples de deux (1024 = 2×2×2×2×2×2×2×2×2×2) aux multiples de dix.<br/></p>
